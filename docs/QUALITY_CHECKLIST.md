# Final Quality Checklist

Use this checklist before submitting or judging the repository.

## Correctness

- [ ] `anchor build` succeeds with Anchor `0.31.1`.
- [ ] `npm run typecheck` succeeds.
- [ ] `npm run test:unit` succeeds against a local validator.
- [ ] `npm run test:integration` succeeds against a local validator.
- [ ] `npm run test:fork:marginfi` succeeds after starting the printed fork
      validator command.
- [ ] `npm run test:fork:kamino` succeeds after starting the printed fork
      validator command.
- [ ] `npm run test:fork:jupiter` succeeds after starting the printed fork
      validator command.
- [ ] `npm run test:fork:maple` succeeds after starting the printed fork
      validator command and using the syrupUSDC fixture generated by
      `npm run clone:mainnet -- maple`.
- [ ] `npm run test:fork:drift` succeeds after starting the printed fork
      validator command and records request-remove withdrawal state.
- [ ] No adapter returns success for a fake protocol integration.
- [ ] Maple is tested as a syrupUSDC asset-position adapter and does not claim
      to perform direct CCIP mint/redeem.
- [ ] Any skipped tests include an explicit integration blocker.

## Interface Design

- [ ] Standard surface remains `deposit`, `withdraw`, `current_value`.
- [ ] Dispatcher validates registry entry PDA, adapter status, adapter program,
      mint, and nonzero amount.
- [ ] Adapters validate owner, mint, and protocol accounts before CPI.
- [ ] `current_value` returns native supported-mint units as `u64`.

## Documentation

- [ ] `docs/SPEC.md` explains the public standard.
- [ ] `docs/BUILD_ADAPTER.md` is usable by a new adapter team.
- [ ] `docs/ARCHITECTURE.md` explains boundaries and flow.
- [ ] `docs/DEPLOYMENT.md` includes local, fork, and devnet commands.
- [ ] `docs/INTEGRATION_NOTES.md` lists every unresolved integration blocker.
- [ ] README includes setup, build, tests, fork tests, devnet deployment, adapter
      status, limitations, and bounty checklist.

## Repository Hygiene

- [ ] Generated ledgers and fixtures are ignored.
- [ ] No private keypairs are committed.
- [ ] No mainnet protocol account or CPI layout is invented.
- [ ] Every placeholder is searchable with `TODO_INTEGRATION:`.
- [ ] SDK examples load IDLs from `target/idl` rather than committing generated
      artifacts.

## Devnet Readiness

- [ ] `npm run deploy:devnet:plan` prints expected commands.
- [ ] `npm run deploy:devnet` builds, deploys the registry, and initializes the
      registry PDA when the Anchor/Solana CLIs and authority keypair are present.
- [ ] The registry governance authority is recorded and secured.

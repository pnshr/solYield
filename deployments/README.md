# Deployments

This directory holds **deployment manifests** — JSON records of real, on-chain
deployments of the registry (and, later, adapters). A manifest is an artifact of
an actual deploy; it is generated by tooling from live chain state, not authored
by hand.

## Files

- `devnet/registry.example.json` — the manifest **schema**, with placeholder
  values. Committed as documentation. It is NOT a real deployment.
- `devnet/registry.json` — the **real** devnet registry manifest. Created by
  `npm run deploy:devnet:registry` (after a successful `anchor deploy` + registry
  initialization) and/or by `npm run verify:devnet:registry -- --write`. This
  file only exists once a genuine devnet deployment has been performed.

## Manifest schema (`devnet/registry.json`)

| Field | Type | Meaning |
| --- | --- | --- |
| `cluster` | string | Always `"devnet"` here. |
| `rpcUrl` | string | RPC endpoint used for the deploy/verify. |
| `registryProgramId` | base58 | Deployed registry program id. |
| `registryConfig` | base58 | `[b"registry_config"]` PDA under the registry program. |
| `governanceAuthority` | base58 | On-chain registry governance authority. |
| `adapterCount` | number | `RegistryConfig.adapter_count` read from chain. |
| `programDataAddress` | base58 \| null | Upgradeable-loader ProgramData account. |
| `programDataSlot` | number \| null | Slot the program was last (re)deployed at. |
| `lastVerifiedSlot` | number \| null | Slot at which verification last ran. |
| `initSignature` | string \| null | Tx signature of `initialize_registry`, if known. |
| `deployedAt` | string \| null | ISO-8601 timestamp of first deploy. |
| `verifiedAt` | string | ISO-8601 timestamp of the last manifest write. |
| `notes` | string | Free-form provenance note. |

## How to reproduce / verify

Anyone can independently confirm the deployment with read-only RPC calls:

```sh
# Reads deployments/devnet/registry.json and checks it against live devnet:
#  - registry program account exists and is executable
#  - program is owned by the BPF upgradeable loader
#  - registry_config PDA exists and is owned by the registry program
#  - manifest program id / config PDA / governance authority match chain
npm run verify:devnet:registry
```

Exit code is non-zero if any check fails. Add `-- --write` to (re)generate the
manifest from the current on-chain state (only succeeds if `registry_config`
already exists on-chain — the script never sends a transaction).
